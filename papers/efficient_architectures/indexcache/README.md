# IndexCache: Accelerating Sparse Attention via Cross-Layer Index Reuse
arXiv: 2603.12201 | Yushi Bai et al. (Tsinghua University & Z.ai, 2026)
Code: https://github.com/THUDM/IndexCache

## Files

- `s01_introduction.md` — Covers the motivation and top-level claim: in DeepSeek Sparse Attention (**DSA**), core attention drops from $O(L^2)$ to $O(Lk)$, but each layer's lightning indexer still performs an $O(L^2)$ selection pass. The paper observes strong cross-layer top-$k$ stability and proposes **IndexCache**, which keeps a small set of Full (`F`) layers and lets Shared (`S`) layers reuse the most recent retained index set. The introduction reports the headline result: removing 75% of indexer computations on a 30B DSA model yields up to **1.82×** prefill speedup and **1.48×** decode speedup with negligible quality loss, with positive preliminary evidence on 744B `GLM-5`.

- `s02_preliminary.md` — Explains DSA's two-stage selection/computation design, where a lightweight ReLU-gated lightning indexer selects $k=2048$ tokens and sparse core attention attends only to that subset. It also frames the key empirical prerequisite for IndexCache: adjacent layers select highly overlapping token sets, echoing prior cross-layer stability findings but now applied to sparse-attention indexer outputs rather than full-attention oracle scores. This section clarifies why prior anchor-layer sparse decoding methods cannot be directly reused for DSA, because DSA has removed full attention from normal inference.

- `s03_method.md` — Details the IndexCache pattern representation, the inference-time `F`/`S` layer behavior, and the two optimization routes. **Training-free IndexCache** starts from all-F and greedily flips layers to `S` by minimizing LM loss on a fixed calibration set, avoiding brittle uniform interleaving. **Training-aware IndexCache** trains each retained indexer with a multi-layer distillation loss against the attention distributions of all layers it serves; the paper proves this has the same gradient as distilling against the averaged target distribution, making the retained indexer learn a consensus top-$k$ set.

- `s04_experiments.md` — Records the main empirical results on a 30B `GLM-4.7-Flash`-based DSA model and a 744B `GLM-5` scaling experiment. At 200K context, `IndexCache (1/4)` reduces prefill latency from **19.5s** to **10.7s**, improves single-request decode from **58** to **86 tok/s**, and raises full-KV-cache throughput from **197** to **297 tok/s**. Greedy training-free search recovers Long Avg from the poor 1/4-uniform value **43.0** to **49.9** versus **50.2** for original DSA, while training-aware 1/4 uniform IndexCache remains within about **0.4** aggregate points of its DSA baseline.

- `s05_related_conclusion.md` — Places IndexCache among training-free sparse attention, trainable sparse attention, hybrid attention architectures, and cross-layer sharing methods. The key distinction is that previous cross-layer top-$k$ reuse methods rely on full attention at anchor layers, whereas IndexCache reuses DSA's cheaper lightning-indexer output and introduces loss-based pattern search plus cross-layer distillation. The conclusion argues that as sparse attention becomes default in frontier LLMs such as `DeepSeek-V3.2` and `GLM-5`, cross-layer index reuse is likely to become a standard inference-efficiency component.

- `s06_references.md` — Full 56-entry reference table covering efficient attention, sparse attention, KV-cache sharing, long-context benchmarks, DSA, `GLM-5`, `DeepSeek-V3.2`, and related cross-layer reuse work.

- `s07_appendix.md` — Preserves the appendices: pairwise top-$k$ overlap analysis, the searched `F`/`S` patterns for the 30B model and `GLM-5`, the negative result for similarity-based dynamic-programming pattern search, and evaluation setup details. The appendix is important because it explains why visually high overlap clusters are not sufficient: local similarity can miss critical-token errors that compound through downstream layers, while greedy LM-loss search optimizes the end-to-end effect.
