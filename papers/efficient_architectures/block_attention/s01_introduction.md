# 1. Introduction

Retrieval-Augmented Generation (**RAG**) is a key technique for mitigating **knowledge hallucination** in LLMs. By retrieving documents from an external database, LLMs can ground their answers in relevant sources. However, in practical scenarios, ensuring recall of the "correct knowledge" often requires retrieving multiple documents (5 to 30) and concatenating them into the input prompt. This causes severe degradation in inference efficiency due to increased input length, with **time-to-first-token (TTFT)** being significantly higher than non-RAG LLM usage.

A natural idea is to **cache the KV states** of documents that have been computed before. However, the KV states of autoregressive LLMs are context-dependent — the same document in different contexts will have different KV states, requiring full re-encoding for every new query.

This paper proposes **Block-attention**, which reduces RAG-LLM TTFT and compute FLOPs to nearly the level of non-RAG LLMs while fully preserving accuracy. The core idea: divide the entire input sequence into multiple **blocks**. Each block independently computes its KV states via self-attention without attending to other blocks, except for the last block (the user query), which can attend to all preceding blocks (retrieved documents). In RAG, each document is a separate block whose KV states can be cached and reused.

Implementation steps: 1) separately encode all blocks except the last; 2) compute position encodings according to their positions in the input prompt; 3) integrate these blocks to compute the final block's KV states. The main challenge is that LLMs have never seen this attention pattern during training — a direct switch causes drastic performance drops (e.g., `Llama-3.1-Tulu-3-8B-SFT` average accuracy drops from **66.1% to 49.9%** on four RAG benchmarks). The solution is **block fine-tuning**, which recovers full accuracy (from 49.9% to **70.0%**) in about 500–1000 steps.

Comprehensive evaluation on **11 benchmarks** (RAG, ICL, and general domains) shows that after fine-tuning, Block-attention models achieve comparable average accuracy to full-attention models. On efficiency: with 50-token user input and increasing total sequence length, Block-attention's advantage grows dramatically — at total length 32K, TTFT and FLOPs-TFT are only **1.3%** and **0.2%** of full attention.
