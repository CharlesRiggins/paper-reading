# Self-Compacting Language Model Agents
OpenReview: 18449 | Anonymous Author(s) (NeurIPS 2026 submission)

## Files

- `s01_introduction.md` — Covers the abstract and full Introduction. Proposes SELFCOMPACT, a training-free scaffold pairing an inline compaction tool with a lightweight rubric that decides when to fire (sub-task resolved, trajectory converging) and when to suppress (mid-derivation, stuck). Key results: up to 18.1 points improvement on competition math, 5–9 points on agentic search, at 30–70% lower per-question token cost. The rubric alone cannot act; the tool alone is used unevenly—both are needed.
- `s02_preliminaries.md` — Covers context management in long-horizon agents (concrete scale numbers from `Qwen3-Coder-Next` at 8M tokens, `Kimi-K2.5` at 96k thinking tokens, `BrowseComp` at hundreds of tool calls) and current compaction strategies. Contrasts reactive compaction (overflow prevention) with periodic compaction (fixed interval), showing their symmetric failure modes: reactive waits too long, periodic discards indiscriminately.
- `s03_selfcompact_approach.md` — The core method section. Details the summarization tool $\mathcal{S} : (x, y_{1:t}) \mapsto \tilde{y}$, the model $\pi$, and SELFCOMPACT's design pairing the tool with a rubric. Preserves complete Algorithm 1 pseudocode, the summarizer design with KV-cache reuse, the rubric triggering mechanism with COMPRESS/CONTINUE verdicts, the three-regime workflow comparison (no-compaction, fixed-interval, rubric-gated), and cost analysis showing $\mathcal{O}(L^2)$ avoidance and $\mathcal{O}(L) \to \mathcal{O}(\ell)$ attention reduction.
- `s04_experiments.md` — Experiments on competition math (IMO-Answerbench, HMMT Nov 25 / Feb 26) across four Qwen reasoning models and agentic search (BrowseComp, BrowseComp-Plus, DeepSearch QA) across three deployed agents (`MiniMax-M2.5`, `GLM-4.7-Flash`, `MiMo-V2-Flash`). SELFCOMPACT leads in 11/12 math settings with gains up to +18.1 points (`Qwen3.5-9B`, HMMT Feb 26) and achieves +5.3 to +9.2 point gains on search at 30–70% lower cost. Includes oracle analysis showing 40.4% degradation from fixed-interval compaction and a 52.9% oracle ceiling.
- `s05_06_07_related_work_limitations_conclusion.md` — Combines Related Work (frontier model compaction, post-training compaction, KV cache eviction), Limitations (model capability gap across open-weight models, no RL training), and the Conclusion framing when-to-compact as a meta-cognitive capability that scaffolds can supply without training.
