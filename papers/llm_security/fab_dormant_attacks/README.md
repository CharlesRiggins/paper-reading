# Watch your steps: Dormant Adversarial Behaviors that Activate upon LLM Finetuning

arXiv: 2505.16567 | Thibaud Gloaguen et al. (ETH Zurich, 2025)
Code: — (included with submission)

## Files

- `s01_front_matter.md` — Title, authors, abstract, and the Figure 1 threat-model overview. Introduces FAB (Finetuning-activated Adversarial Behaviors), a meta-learning attack that plants dormant adversarial behaviors into an LLM, activated only when a downstream user finetunes the model on benign datasets.
- `s02_introduction.md` — Motivates the attack by challenging the assumption that finetuning is a controlled, secure process. Summarizes the three attack scenarios (advertisement injection, jailbreak, over-refusal) and lists the main contributions, including robustness across user finetuning configurations.
- `s03_background_related_work.md` — Covers jailbreaking attacks, backdoor attacks, model finetuning safety, and meta-learning. Emphasizes the key novelty: no prior work uses the finetuning process itself as a trigger, which removes the adversary's need to know the victim's dataset.
- `s04_fab_method.md` — Details the FAB method: the threat model, Algorithm 1, and the three loss components — first-order meta-learning ($\mathcal{L}_{\text{m-l}}$, Equation 1–2), noise-based robustness ($\mathcal{L}_{\text{noise}}$, Equation 3), and utility regularization ($\mathcal{L}_{\text{reg}}$, Equation 4). The attacker jointly optimizes these to keep the model benign before finetuning while ensuring adversarial activation after.
- `s05_evaluation.md` — Full empirical evaluation across three attack scenarios with all 9 result tables. Advertisement injection reaches up to **65.3% ASR**, jailbreak achieves over **8× higher rates** (surpassing 90% ASR), and over-refusal renders models useless in up to **25%** of cases. Robustness ablations (Table 7) show the noise component yields a **2.5× average ASR increase**; component ablations (Table 8) confirm meta-learning + noise is essential; DPO and distillation also trigger the attack (Table 9).
- `s06_conclusion_ethics_reproducibility.md` — Concludes that finetuning is a novel attack vector, discusses mitigations (post-finetuning security evaluation), and notes the higher computational cost for adversaries. Ethics statement argues awareness is key to defense; reproducibility statement confirms code release.
- `s07_references.md` — Full 51-entry reference list spanning meta-learning (Finn et al., Nichol et al.), LLM safety (Qi et al., Halawi et al., Tamirisa et al.), backdoor attacks (Rando & Tramèr, Kurita et al.), and evaluation benchmarks.
- `s08_appendix_a_experiments.md` — Full ASR curves for all main results and ablation experiments (Figures 3–34), plus Table 10 validating that the user finetuning configuration produces meaningful model changes on paired benchmarks.
- `s09_appendix_b_details.md` — Complete hyperparameters for FAB training, instruction-tuning, and trigger-finetuning across all attack scenarios. Includes per-scenario dataset compositions for $\mathcal{D}_{\text{adv}}$ and $\mathcal{D}_{\text{reg}}$, benchmark evaluation protocol, judge details, and runtime table showing linear scaling of training time with meta-learning steps $k$ (1h08 for $k=1$ to 6h04 for $k=100$).
- `s10_appendix_c_d_broader_impact.md` — Prompts and templates (Appendix C) plus broader impact (Appendix D): advocates for post-finetuning security evaluation and community-driven reporting, acknowledges limitations (restricted to ≤3B models due to meta-learning cost), and details computational resources (single H100/GH200, 3h30m average FAB training for `Llama-3.2-1B`).
