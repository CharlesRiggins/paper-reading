# Group-in-Group Policy Optimization for LLM Agent Training
arXiv: 2505.10978 | Lang Feng et al. (Nanyang Technological University, Singapore / Skywork AI, Singapore, 2025)
Code: [github.com/langfengQ/verl-agent](https://github.com/langfengQ/verl-agent)

## Files

- `s01_introduction.md` — Introduces the credit assignment challenge in multi-turn LLM agent training and proposes **GiGPO**, a two-level group-based RL algorithm that combines episode-level macro advantages with step-level micro advantages via **anchor state grouping**. Unlike prior group-based RL (GRPO, RLOO) limited to single-turn tasks, GiGPO retroactively identifies repeated environment states across trajectories to construct step-level groups for fine-grained per-step credit signals, achieving >12% gains on ALFWorld and >9% on WebShop over GRPO with negligible (<0.002%) extra time cost.
- `s02_related_work.md` — Surveys three research threads: LLMs as decision-making agents (ReAct, Reflexion, tool use), RL for LLM agents (DQN, PPO, ArCHer, AgentQ, LOOP, RAGEN), and RL for LLM reasoning (RLOO, GRPO, DAPO, CPPO). Positions GiGPO as orthogonal to existing group-based RL advances, retaining their critic-free efficiency while introducing hierarchical credit assignment specifically for agent training.
- `s03_preliminaries.md` — Formalizes the multi-turn agent RL setting: LLM policy $\pi_\theta(\bm{a}_t|\bm{s}_t,x)$ generates textual actions over $T$ steps in an environment with sparse/delayed rewards. Recaps group-based RL where $N$ trajectories are sampled and advantages are computed via group statistics instead of value functions, using GRPO's normalization-based advantage as the canonical example.
- `s04_method.md` — Presents GiGPO's two-level advantage structure in detail: (1) **Episode relative advantages** $A^E$ normalize trajectory returns against the group mean (with std or $F_{\text{norm}}=1$ for unbiased RLOO-style estimation); (2) **Step relative advantages** $A^S$ use anchor state grouping to retroactively match identical environment states across trajectories via hashmap lookups, compute discounted returns, and compare actions per-state. The combined group-in-group advantage $A = A^E + \omega A^S$ feeds into a clipped PPO-style objective with KL regularization.
- `s05_experiment.md` — Evaluates GiGPO on ALFWorld, WebShop, and search-augmented QA with Qwen2.5-1.5B/3B/7B-Instruct. GiGPO surpasses GRPO by 13.3% on ALFWorld and 10.6% on WebShop at 1.5B, achieves 42.1% (3B) and 47.2% (7B) on search QA. Ablation confirms both $A^E$ and $A^S$ are essential. Step-level group dynamics show convergence from diverse to concentrated (6–8) groups as training progresses. The additional GiGPO components add only 0.54s per iteration vs. 362.83s baseline (<0.002% overhead).
- `s06_conclusions.md` — Summarizes GiGPO as a hierarchical group-based RL algorithm enabling fine-grained per-step credit assignment for LLM agents without extra rollout or memory cost. Notes a limitation: state matching for anchor construction may fail in noisy environments with subtle state differences, though GiGPO degrades gracefully to GRPO. Future directions include embedding-based state matching strategies.
- `s07_references.md` — Full 78-entry reference list covering LLMs, RL for agents, group-based RL, benchmarks, and training frameworks.
- `s08_appendix.md` — Covers the verl-agent open-source framework with step-wise multi-turn interaction and broad environment/algorithm support; mathematical proof of unbiasedness for $F_{\text{norm}}=1$; full pseudo-code; detailed hyperparameters for all three task settings; prompt templates; VLM agent results on Sokoban (81.0%) and EZPoints (100%); orthogonality to DAPO yielding 87.5 WebShop score; sensitivity analysis showing robustness to $\omega \in [0.4, 1.2]$; and a full reasoning trajectory example.
