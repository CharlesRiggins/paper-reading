# DAPO: An Open-Source LLM Reinforcement Learning System at Scale
arXiv: 2503.14476 | Qiying Yu et al. (ByteDance Seed / Tsinghua AIR / HKU, 2025)
Code: [github.com/volcengine/verl](https://github.com/volcengine/verl) | Project: [dapo-sia.github.io](https://dapo-sia.github.io/)

## Files

- `s01_introduction.md` — Presents the motivation for open, reproducible large-scale LLM RL in long-CoT reasoning. The paper starts from the gap between closed frontier systems such as OpenAI o1 and DeepSeek R1 and the community’s difficulty reproducing their RL results, then shows that naive `GRPO` on `Qwen2.5-32B` reaches only **30** on AIME. DAPO is introduced as an open-source RL system that reaches **50** on AIME 2024, surpassing `DeepSeek-R1-Zero-Qwen-32B` (**47**) with about **50%** of the training steps.
- `s02_preliminary.md` — Recaps the optimization background: PPO’s clipped surrogate objective, GAE, GRPO’s group-relative advantage normalization, and the sample-level reduction used by GRPO. It also explains why DAPO removes the KL penalty in long-CoT RL and uses rule-based final-answer rewards instead of a learned reward model to avoid reward hacking on verifiable math tasks.
- `s03_dapo.md` — Details DAPO’s core algorithm and four recipe-level fixes. **Clip-Higher** decouples lower and upper clipping with $\varepsilon_{\mathrm{low}}=0.2$ and later $\varepsilon_{\mathrm{high}}=0.28$ to reduce entropy collapse; **Dynamic Sampling** filters prompt groups with all-correct or all-wrong responses; **Token-Level Policy Gradient Loss** reweights long CoT sequences by token rather than sample; and **Overlong Reward Shaping** adds soft length punishment to reduce truncation-induced reward noise. The file also covers `DAPO-Math-17K`, the integer-answer math dataset built to make rule rewards reliable.
- `s04_experiments.md` — Records the training setup and empirical results. Training uses `Qwen2.5-32B`, `verl`, `AdamW`, learning rate $1\times10^{-6}$, prompt batch size **512**, **16** responses per prompt, max generation length **20,480**, and AIME `avg@32` evaluation. Progressive ablation shows Naive `GRPO` at **30**, Overlong Filtering at **36**, Clip-Higher at **38**, Soft Overlong Punishment at **41**, Token-level Loss at **42**, and full DAPO at **50**. The section also explains training-dynamics monitoring and reflective behavior emerging during RL.
- `s05_conclusion_contributions.md` — Summarizes DAPO as a fully open-sourced large-scale LLM RL system consisting of algorithm, infrastructure, and dataset. It preserves the paper’s contribution roles, including project lead, algorithm, infrastructure, dataset, supervision, affiliations, and acknowledgments to supporting ByteDance colleagues.
- `s06_references.md` — Full **39-entry** reference list covering o1, DeepSeek-R1, GPT-4, Claude, Qwen, Kimi, reproduction attempts, RLHF, PPO, GRPO/DeepSeekMath, `verl`/HybridFlow, reward hacking, theorem proving, coding RL, and `AdamW`.
- `s07_appendix.md` — Captures the appendix examples. The dataset-transformation example rewrites an answer of $11-2\sqrt{6}$ into the integer target $k+m+n=19$, illustrating how `DAPO-Math-17K` makes rule-based reward parsing reliable. The supplementary case shows reflective behavior: the model detects that a combinatorial count $a_4=54.75$ is impossible and changes strategy rather than continuing blindly.
